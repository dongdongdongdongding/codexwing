# 📈 Swing Trading Bot - 서비스 개요 및 개발 히스토리

본 문서는 이 프로젝트가 어떤 목표를 가지고 만들어진 서비스인지, 그리고 어떤 과정을 거쳐 현재의 고도화된 시스템(V2.5)으로 발전해 왔는지 기록한 문서입니다.

---

## 🎯 1. 서비스 개념 및 목표 (Service Overview)

**"Swing Trading Bot"**은 한국 주식(KOSPI/KOSDAQ) 및 미국 주식 시장을 대상으로, 단기~스윙(3일~10일) 투자에 적합한 종목을 발굴하고 자동 매매 시그널을 제공하는 **AI 기반 퀀트 트레이딩 서비스**입니다.

### 핵심 기능 (Core Features)
1. **투트랙 스캐닝 시스템:**
   - **수동 스캐너 (Streamlit Web UI):** 사용자가 직접 버튼을 눌러 조건(표준/엄격/느슨)에 맞는 주도주를 실시간으로 스캔하고, 분석 결과를 시각적으로 확인합니다.
   - **자동 스캐너 (Auto Bot):** 백그라운드 서버에서 지정된 시간(정규장 오픈 후, 장 마감 전 등)에 전체 시장을 스캔하고, 매수 시그널 발생 시 텔레그램으로 알람을 전송합니다.

2. **다차원 데이터 분석 (Quant Strategy):**
   - **기술적 지표:** MACD, RSI, ATR, 이동평균선(MA), 볼린저 밴드 등 모멘텀과 추세 지표 분석.
   - **수급 및 거래량 (투자자 동향):** 단순히 거래량만 보는 것이 아니라, 외국인/기관 등 메이저 수급의 자금 흐름(VWAP, OBV)을 추적합니다.
   - **매크로 환경:** 시장 상황(코스피/나스닥 지수 흐름, 경제 지표 등)을 반영하여 타점 강도를 조절합니다.

3. **Global Brain AI 예측 모델:**
   - 기존의 단순 조건식 검색기를 넘어, 수십만 개의 과거 캔들 패턴 데이터를 머신러닝(RandomForest + GradientBoosting)으로 학습하여 **"현재 시점에서 매수했을 때 향후 5일 내 상승할 확률"**을 퍼센트(%) 기반으로 예측합니다.

## 🛠️ 2. 개발 및 업데이트 히스토리 (Development History)

단순한 조건식 봇에서 출발하여, 현재의 머신러닝 기반 하이브리드 자동매매 시스템으로 진화하기까지의 주요 개선 기록입니다.

### [Phase 1] 낡은 로직 제거 및 데이터 정합성 확보
- **문제점:** 자동 봇(`auto_bot.py`)과 웹 스캐너(`app.py`)가 서로 오래된 버전(V1)의 Alpha Score 로직을 사용하거나, 수급 데이터를 가짜(단순 거래량 추정)로 사용하고 있어 결과가 불일치하는 심각한 오류가 있었습니다.
- **해결 및 개선:**
  - 두 시스템 모두 실시간 진짜 수급 데이터를 가져오는 `get_investor_flows()`로 데이터 소스를 통일했습니다.
  - 구버전 스코어링 로직을 `calculate_alpha_score_v30()`(V30 통합 로직)으로 교체하여 점수 체계를 완벽히 동기화했습니다.
  - Prophet(시계열 예측) 모델에서 출력 키 오류(`Predicted` -> `yhat`)를 수정하여 예측값이 0으로 나오는 버그를 고쳤습니다.

### [Phase 2] 사용자 경험(UI) 및 필터링 강화
- **문제점:** 스캐너의 필터 기준(전체/기본/강력)이 너무 모호했고, 사용자가 어떤 기준으로 종목이 걸러지는지 알기 어려웠습니다.
- **해결 및 개선:**
  - 직관적인 한글 라벨링 도입: `🔵 표준 (승률40%+, PF 0.8+)`, `🟢 엄격 (승률60%+, PF 1.5+)`, `🟡 느슨 (시그널만)`.
  - 결과창에 **"판정(Verdict)"** 컬럼을 추가하여 AI 확률과 알파 스코어를 종합한 기계의 판정(🟢 강력 매수, 🔵 매수, 🟡 관망)을 명확히 보여주도록 개편했습니다.

### [Phase 3] 🧠 Global Brain V2.5 모델 고도화 (AI 정밀도 혁신)
- **문제점:** 기존 V1/V2 모델은 전체 정확도는 60%대였지만, 중요도가 0인 피처(볼린저 밴드 등)를 사용하고 있어 쓸데없이 매수 시그널을 남발(False Positive)하는 문제가 있었습니다. 또한 5년 전과 지금의 시장 상황이 다름에도 똑같은 가중치로 학습했습니다.
- **해결 및 개선 (V2.5 업데이트):**
  - **피처 가지치기 (Feature Pruning):** 노이즈만 일으키는 `BB_Pct`(볼린저) 지표를 모델에서 완전히 삭제했습니다.
  - **강력한 수급 피처 신설:** `OBV_Slope`(매집 기울기), `VWAP_Dist`(기관 평단가 대비 이격도), `Range_Pct`(변동폭) 등 실전 트레이딩에서 가장 중요한 수급 피처 3가지를 새로 투입했습니다.
  - **최신 트렌드 가중치 (Recency Weighting):** 최근 2년(24~5개월)치 데이터에 가중치(x2)를 줘서 고금리 시대의 최신 장세를 딥하게 반영도록 했습니다.
  - **시계열 교차 검증 (TimeSeriesSplit CV):** 데이터를 무작위로 섞지 않고, 과거에서 미래로 흐르는 주식 특성에 맞춰 5-Fold 검증 시스템을 도입했습니다.
  - **최적 매수 기준선 타겟팅 (Optimal F1 Threshold):** 기계적으로 확률이 50%를 넘어야 산다는 공식을 파괴하고, F1 Score 연산을 거쳐 이번 모델에 최적화된 **"32.7%"** 라는 자체 임계값을 찾아내 스캐너에 이식했습니다.

## 🔮 3. 향후 발전 방향 (Next Steps)
- **자동매매 API 연결:** 스캐너 결과 및 AI 예측이 일정 기준(예: 강력 매수 + 확률 50% 이상)을 초과할 경우 증권사 API를 통해 실주문까지 연동하는 완전 자동화.
- **뉴스/시황 생성형 AI 분석 (News Analysis):** 종목별 실시간 뉴스를 LLM(OpenAI)으로 분석하여 단순 차트를 넘어선 이벤트 드리븐(Event-Driven) 점수 추가 편입 적용.

### [Phase 4] 🛡️ 완전무결 퀀트 엔진 및 전체 시스템 안정성 대규모 고도화 (The Flawless System V32)
- **문제점:** 
  1. 기존 엔진은 매수 시그널 발생 당일 종가(T+0)에 엑셀처럼 곧바로 매수가 체결된다고 가정한 "비현실적 호가(Zero Slippage/Tax)" 시스템이었습니다. 
  2. 오전에 거래량을 일직선(Linear)으로 예측해 잦은 False Positive가 발생했습니다.
  3. `ZeroDivisionError`, `TypeError`, `FinanceDataReader JSON Decode Error` 등 대규모 전수조사 시 각종 시스템 크래시가 발생했습니다.
  4. 과거의 매매 승률보다 최근의 승률이 중요함에도 동일 가중치로 백테스팅되었습니다.
- **해결 및 개선 (V32 업데이트):**
  - **초현실적 백테스팅 (Realism & Costs):** 오늘(T) 시그널을 확인하고 내일 오전 9시 장초방(T+1) 시가에 매수/매도하는 현실적인 룰(Next-Day Open Execution)을 백테스터 전면 최하단에 이식했습니다. 또한 한국장 0.4%, 미국장 0.15%의 확정 수수료/세금을 모든 백테스트 연산에서### Phase 10: U자형 장중 거래량 보정 추가 (Advanced Volume Projection)
*   **문제 원인:** `Total_Mins / Elapsed_Mins`의 단순 선형 곱셈 사용. 장 초반 30분에 하루 거래량의 30%가 터지는 한국 시장의 U자형 생태를 전혀 반영하지 못함.
*   **해결 로직:** `get_intraday_volume_multiplier` 함수 내부에 `is_advanced_engine` (V32) 전용 U-Shape Profile (아침 30%, 점심 40%, 오후 30%) 방식을 적용하여 초반 뻥튀기 현상 완벽 제거.

### Phase 11: AI 급등예측 태그(Surge Tag) 정확도 개선 (Precision 80% 달성 모델 교체)
*   **증상:** V32 스캐너 실시간 라이브 구동 결과, 5개의 `+3%~+5% 가능` 태그 발급 종목 중 대덕전자를 제외한 4개가 모두 고점(Peak)에서 발생한 허위 뱃지(가짜 급등 예측)로 판명됨 (적중률 20%).
*   **원인 규명:** `app.py`에서 `급등예측` 태그를 결정하는 조건식(`_cond_10`, `_cond_5` 등)이 단순히 볼린저 밴드의 중간 이상(`_bb_pos >= 0.3`)이고 MACD가 양수면 통과되는 등 극도로 허술한 규칙(Rule-based)의 나열이었음. 실제 똑똑한 AI 모델(`get_ml_prediction`)의 확률 결과(`prob`)는 태그 배정과 연결조차 되어 있지 않은 상태 방치. 게다가 `RSI`나 `고점` 여부에 따른 과열 구간 방어 로직 아예 제로.
*   **근본 수술 (XGBoost 재학습):** `train_model.py` (Pillar 3 정밀도 모델)의 타겟 변수를 단순히 "3일 내 수익성"에서 **"3일 내 시가 대비 최소 +5% 이상 절대로 튀어오르는가?"** 로 개조하고 KOSPI/KOSDAQ 2,500개 종목을 재학습. (학습 완료 후 Precision: 81.1%).
*   **앱 연결 (Pure AI Mapping):** 앱 내의 쓸데없는 Rule-based 조건식을 전부 폐기처분하고, 새롭게 학습된 `ml_prob` (0~100)을 직접 연동.
    - 확률 90% 이상: `🚀 +10% 가능`
    - 확률 80% 이상: `⬆️ +5% 가능`
    - 확률 70% 이상: `📈 +3% 가능`
*   **고점 2중 방어 하드코딩:** 아무리 확률이 99%가 나와도 **`RSI >= 70`이거나 위치 값이 `고점(Peak)`이면 강제로 배지를 박탈(Suppression)**하도록 `app.py` 검색 로직을 뜯어고쳐, 오늘 디아이나 한스바이오메드 같은 치명적인 매수 실수를 시스템적으로 원천 차단함.

### Phase 12: yfinance 주간(Weekly) 데이터 Fetch 에러 원천 차단 및 주봉 추세 필터 정상화
*   **증상:** 터미널 창에 `The following 'Stock Splits' events are out-of-range, did not expect with interval 1wk` 에러가 발생하며 멀쩡한 코스닥 종목들(예: 코퍼스코리아 009730)이 스캔에서 누락됨.
*   **원인:** `yfinance` 라이브러리가 `interval="1wk"` 로 주간 데이터를 호출할 때, 액면분할 날짜가 특정 주간에 겹치면 파싱 에러를 일으켜 전체 데이터 다운로드를 중단시키는 치명적인 고질적 버그가 존재함.
*   **해결:** `quant_analysis.py`에서 굳이 API를 한 번 더 쏴서 `self.df_weekly`를 받아오던 불필요한 코드를 전면 삭제. 어차피 일봉(`self.df`)을 `resample('W')` 하여 주봉 데이터를 수학적으로 자체 합성해 오고 있었으므로, 주봉 추세(Weekly Trend) 판독 필터를 기존 API 의존 방식에서 **자체 리샘플링 데이터(`self.df['W_Trend']`) 연결 방식으로 수정**.
*   **추가 효과:** 쓸데없던 주봉 API 호출을 삭제하여 **스캐너 전체 속도 2배 향상** 및 주간/일간 추세 필터 정상화 완료.

### Phase 13: 대장주/주도주 하이패스 (Market Leader Bypass) 알고리즘 도입
*   **증상:** SK이터닉스, 주성엔지니어링, 우리로 같은 폭발적인 모멘텀 대장주들이 "쩌리" 종목들에 밀려 스캐너에 전혀 노출되지 않는 현상 발생.
*   **원인:** 스캐너의 기존 로직이 백테스트 수익성(T+1 시가 매수, 2일 뒤 매도)을 기계적으로 매우 엄격하게 체크(`wr < 40 and pf < 0.8: pass_filter = False`)하고 있었음. 대장주 돌파매매는 그 특성상 변동성이 너무 커 과거 데이터로 기계적인 2일 보유 백테스트를 돌릴 경우 승률이 40% 미만으로 나오는 경우가 잦아, **이 위대한 종목들이 내부 필터에서 모조리 광탈(Drop)** 당하고 있었음.
*   **해결 로직:** `app.py`에 `is_market_leader` 변수 신설. 
  - AI 급등 확률(ml_prob) >= 65% 이면서
  - 당일 거래량이 20일 평균 대비 1.5배 이상 폭발하고
  - 당일 양봉 마감(`Close > Open`) 인 종목이라면
  - **과거의 기계적 백테스트 승률(WR)과 수익팩터(PF)가 낮더라도 모든 제약을 강제 돌파(하이패스)하여 화면에 출력**하도록 필터 체계를 전면 개편.
*   **UI 피드백:** 이 방식으로 구제되어 튀어나온 종목들은 화면 전략 태그에 `👑 주도주 하이패스` 라는 극강의 뱃지를 달도록 디자인 로직 추가.

---:** 과거 매매 횟수가 3회 미만인 종목의 승률표에 엄격한 페널티를 부과하고, 2년 전보다 최근에 발생한 매매 승률에 가중치(Recency Weighting)를 부여해 현재 장세에 맞는 승률만 필터링하도록 로직을 고도화했습니다.
  - **대규모 스캔 에러 원천 차단 (Robustness):** 
    - `464080.KQ` 등 백테스트 연산 중 진입가(Entry)가 0원으로 잡힐 때 터지던 `ZeroDivisionError` 버그 완벽 패치.
    - 웹 UI에서 종목명이 누락되었을 때 터지던 `TypeError: NoneType` 스트림릿 앱 다운 에러 방어 코드 핫픽스 추가.
    - 코스피/코스닥 2500개 전수조사 시 발생하는 `FinanceDataReader`의 JSON 디코딩 에러 및 야후 파이낸스 API `Rate Limit(429)` 이슈를 자체 내부 티커 파싱 로직과 병렬 멀티스레딩(Multi-threading) 최적화로 돌파.
  - **필터 아키텍처 다원화 및 역할 규명:**
