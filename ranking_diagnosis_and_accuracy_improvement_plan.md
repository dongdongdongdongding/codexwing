## 2. 현재 `Decision Score`가 실제로 무엇에 끌리는가

결과 파일 기준으로 `Decision Score`와 다른 변수의 관계를 보면 다음 경향이 있다.

- `Antigrav`와의 상관: **0.546**
- `급등예측 확률(surge_prob)`과의 상관: **0.426**
- `거래량 확인(volume_ok)`과의 상관: **0.210**
- `승률`과의 상관: **0.164**
- `수급 점수`와의 상관: **0.002**
- `UP 추세`와의 상관: **-0.096**
- `고점(Peak)` 여부와의 상관: **-0.229**
- `단기과열` 여부와의 상관: **-0.306**

### 해석
현재 랭킹은 대체로:
1. **Antigravity Score**
2. **급등예측 확률**
에 많이 끌리고,

상대적으로:
- 수급 점수
- 추세 방향
- 위치(고점/조정/상승)
- 거래량 확정성

은 최종 순위에서 충분히 강하게 반영되지 않는 것으로 보인다.

즉 지금 구조는:

> **“좋은 종목”보다 “높은 확률/높은 점수”에 과체중된 정렬**

에 더 가깝다.

## 3. 현재 랭킹식의 핵심 문제

### 3-1. 확률과 매매 자리의 질이 분리되지 않음
현재 상위권에는 다음 유형이 함께 보인다.

- 추세가 DOWN인데 확률이 높게 나온 종목
- 거래량이 약한데 점수가 높은 종목
- 이미 너무 많이 올라서 Peak에 가까운 종목
- 단기과열 경고가 붙었는데도 상위권에 남는 종목

즉 현재 랭킹은 **“될 가능성”**은 반영하지만,  
**“지금 사도 되는 자리인가”**는 아직 충분히 반영하지 못한다.

### 3-2. `Decision Score`가 리스크를 충분히 패널티하지 않음
지금 결과를 보면 아래 리스크는 아직 순위에 약하게 반영되는 것으로 보인다.

- `Peak` 위치
- `단기과열`
- `DOWN` 추세
- 거래량 미확인
- T3 등급

이 상태면 상위권이 **실전 진입 우선순위**와 어긋날 수 있다.

## 4. 현재 결과를 기준으로 본 개선 방향

## 4-1. `Decision Score`에 리스크 패널티를 직접 넣기
가장 먼저 해야 할 일은 최종 정렬식에 아래 항목을 **직접 패널티**로 넣는 것이다.

### 필수 패널티
- `Peak` 패널티
- `단기과열` 패널티
- `DOWN 추세` 패널티
- 거래량 `⚠️` 패널티
- T3 패널티

### 필수 보너스
- `UP 추세` 보너스
- `Rising` 위치 보너스
- 거래량 `✅` 보너스
- T1 / T2 보너스
- 수급 60점 이상 보너스

### 예시 수식
```python
decision_score_v2 = (
    base_prob_score
    + antigravity_score * 0.35
    + whale_score * 0.10
    + trend_bonus
    + volume_bonus
    + tier_bonus
    - peak_penalty
    - overheat_penalty
    - downtrend_penalty
)
```

---

## 4-2. `probability`와 `trade quality`를 분리하기
지금은 확률 모델이 너무 많은 역할을 한다.  
앞으로는 다음 둘을 나눠야 한다.

### 1단계: target hit probability
- `P(+3%)`
- `P(+5%)`
- `P(+10%)`

### 2단계: trade quality / execution quality
- `clean hit` 확률
- `MAE` 위험
- `time_to_target`
- `expected_return / expected_drawdown`

즉 앞으로는:
- **될 가능성**
- **지금 들어가도 되는 질 좋은 자리인가**

를 따로 판단해야 한다.

---

## 5. 예측 정확도를 높일 수 있는 실질 개선안

## 5-1. 현재 가장 효과가 클 개선: Meta-Quality Model 추가
현재 스캐너는 “오를 가능성”은 어느 정도 잘 본다.  
하지만 실제 매매 만족도는 아래가 더 중요하다.

- 바로 가는가
- 흔들리지 않고 가는가
- 손절 먼저 맞을 위험은 낮은가
- 너무 늦은 자리가 아닌가

### 추천
별도 모델을 하나 더 둔다.

- **1차 모델:** target hit probability
- **2차 모델:** trade quality probability

예:
- `P(clean +5% hit before ugly drawdown)`
- `P(fast hit within 3~5 days)`
- `P(no stop sweep before target)`

이 모델이 붙으면 상위권 품질이 눈에 띄게 좋아질 가능성이 크다.

---

## 5-2. 레짐별 모델 분기
현재 시장이
- 상승장
- 반등장
- 하락장
- 테마장
- 저변동 박스장

인지에 따라, 같은 패턴의 의미가 달라진다.

### 개선안
시장 레짐을 먼저 분류한 뒤:
- 상승장용 `P(+5%)`
- 하락장용 `P(+5%)`
- 테마장용 `P(+10%)`

처럼 조건부 모델로 나누는 것이 좋다.

### 기대 효과
- DOWN 추세 종목의 과대평가 감소
- 강한 모멘텀장에서는 진짜 리더주 포착력 상승
- 박스장에서는 과열 추격 감소

이 단계는 향후 정확도 개선 폭이 가장 클 가능성이 높다.

---

## 5-3. 라벨 고도화: 성공/실패만 보지 말고 “질”을 나누기
현재도 라벨 구조는 꽤 좋다.
- T+1 Open
- 비용 반영
- stop first
- pessimistic fill

다만 이제는 성공도 아래처럼 나눌 필요가 있다.

### 추천 라벨
- `clean_hit`
- `dirty_hit`
- `late_hit`
- `fast_hit`
- `peak_chase_failure`

이렇게 하면:
- 뒤늦은 종목
- 너무 흔들리는 종목
- 보기만 좋은 종목

을 훨씬 잘 걸러낼 수 있다.

---

## 5-4. 순위화는 `Threshold + Top-K` 하이브리드가 좋다
현재 실험에서 `Threshold >= 60%`가 top ranking보다 precision은 좋았다고 해도,  
실전 운영은 아래 구조가 더 좋다.

### 추천
1. 최소 신뢰 임계치 통과
   - 예: `prob_5 >= 0.58`

2. 그 안에서 다시 Top-K 정렬
   - 예: `decision_score_v2` 기준 상위 5개

이렇게 하면:
- pure threshold의 기회 부족
- pure ranking의 억지 선택

을 동시에 줄일 수 있다.

---

## 5-5. 랭킹에 `경로 리스크`를 넣어라
정확도를 높이려면 앞으로는 단순 hit-rate보다 아래가 중요하다.

- `expected_MAE`
- `expected_time_to_target`
- `overnight gap risk`
- `stop-hit risk`

### 추천 방향
`decision_score`를 아래처럼 바꾼다.

```python
decision_score_v3 = (
    calibrated_prob_5
    * expected_return_5
    / max(expected_drawdown, 0.01)
)
```

또는:

```python
decision_score_v3 = (
    hit_prob
    * clean_hit_prob
    * trend_quality
    * volume_confirmation
)
```

즉, 앞으로는 **확률 × 기대값 × 리스크조정**으로 가야 한다.

## 6. 실전형 정렬 예시

현재 결과를 보면 아래 원칙이 더 실전적이다.

### 올려야 하는 종목
- UP 추세
- Rising 위치
- 거래량 `✅`
- T1 / T2
- 수급 60+
- 단기과열 아님

### 내려야 하는 종목
- DOWN 추세
- Peak 위치
- 거래량 `⚠️`
- 단기과열
- T3
- 전일비가 이미 과도하게 급등한 종목

### 예시 룰 기반 보정
```python
rank_adjust = 0

if trend == "UP":
    rank_adjust += 6
if "Rising" in position:
    rank_adjust += 5
if volume_confirmed:
    rank_adjust += 4
if tier in ["🏆T1", "⭐T2"]:
    rank_adjust += 5
if whale_score >= 60:
    rank_adjust += 3

if trend == "DOWN":
    rank_adjust -= 8
if "Peak" in position:
    rank_adjust -= 10
if "단기과열" in strategy:
    rank_adjust -= 8
if not volume_confirmed:
    rank_adjust -= 5
if tier == "T3":
    rank_adjust -= 4

decision_score_v2 = decision_score + rank_adjust
```

이 정도만 해도 지금보다 상위권 품질이 훨씬 자연스러워질 가능성이 높다.

## 최종 결론

현재 스캐너는 **종목을 못 찾는 게 아니라, 오히려 꽤 잘 찾는다.**  
문제는 최종 상위 랭킹이 아직 **“좋은 매매 자리”보다 “높은 숫자 후보”에 약간 치우친다**는 점이다.

가장 정확한 결론은 다음과 같다.

> **현재 시스템은 후보 발굴 엔진으로는 합격이다.**
>
> **예측 정확도를 더 높이려면 이제 기술지표를 더 붙이는 것보다,**
> **확률과 실행 품질을 분리하고, 최종 정렬식에 리스크 패널티를 넣는 방향으로 가야 한다.**

한 줄 요약:

> **발굴은 잘한다. 이제 순위화를 실전형으로 바꾸면 정확도 체감이 훨씬 좋아진다.**
